Single period geometric attribution

Anand Krishnakumar

2018-04-26

This vignette shows how to perform a single period geometric attribution analysis using the PortfolioAttribution package and illustrates the point about the weight inputs being treated as end-of-period weights. For a brief introduction to portfolio performance attribution, this model, the package and references, please refer to the introduction vignette.

Define the data inputs

Say we have a relatively large portfolio & benchmark data set at the security level. This example data set in the package has 2958 assets over 13 monthly periods. The column names are security identifiers and the row indexes are month-end dates. As outlined in the introduction vignette, the attribution package requires that the number of assets be the same for all periods and between the portfolio & benchmark.

The first few periods & assets are shown below.

To perform a single period attribution analysis just the first observation for the portfolio & benchmark is extracted. Since the package uses an end-of-period weight convention, the index of the weight data is the month-end prior to that of the return data.

wp = multi_period_portf_1$wp[1,]
Rp = multi_period_portf_1$Rp[1,]
index(wp)
#> [1] "2017-01-31"
index(Rp)
#> [1] "2017-02-28"

wb = multi_period_portf_1$wb[1,]
Rb = multi_period_portf_1$Rb[1,]
index(wb)
#> [1] "2017-01-31"
index(Rb)
#> [1] "2017-02-28"

Geometric attribution

To run geometric attribution analysis on this data the following command is used.

attribution_results = Attribution(Rp, wp, Rb, wb, geometric = TRUE)

The total allocation & selection effects for all securities is available in the Total column for the period. In this case, since we had no categories and all the data was at the security level, the selection effects are by construction all zero.

attribution_results$Allocation[1, "Total"]
#> [1] 0.02380793
attribution_results$Selection[1, "Total"]
#> [1] 0

The data set also reports the total portfolio & benchmark return for each period and so the geometric excess return can be computed from that

geometric_excess_return = as.numeric((1+multi_period_portf_1$periodicPortReturns[1,])/
                                       (1+multi_period_portf_1$periodicBmkReturns[1,]) - 1)
geometric_excess_return
#> [1] 0.02380793

With geometric attribution, the total allocation & selection effects compound to give the total effect, which as expected equals the total geometric excess return.

total_effect = (1+attribution_results$Allocation[1, "Total"])*(1+attribution_results$Selection[1, "Total"])-1
all.equal(geometric_excess_return, total_effect)
#> [1] TRUE
